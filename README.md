# 3D_attention_GAT-GCN

## Resources

Original concept and base code from:
https://github.com/shangwenhui/3D_attention_GAT-GCN
- `README.md` : this file.
- `data/train.csv`, `data/val.csv`, `data/test.csv`  
  Training, validation and test splits used for model development.
- `data/processed/train.pt`, `data/processed/val.pt`, `data/processed/test.pt`  
  Preprocessed graph data generated by `3Dcreate_data.py`.

### Source codes
- `3Dcreate_data.py` : converts SMILES strings in CSV files into PyTorch Geometric graphs (3D conformers, atom features, edge distances) and saves them as `.pt` files.
- `utils.py` : contains the `TestbedDataset` class and performance metrics (MSE, RMSE, MAE, Pearson, Spearman, CI, R²).
- `training.py` : **10‑run training script** – runs the same model multiple times with different random seeds, reports mean ± std and 95% confidence intervals for all metrics. Ideal for obtaining robust, statistical performance estimates.
- `training_single.py` : **single‑run training script** – trains one model once, saving the best checkpoint according to validation MSE. Suitable for quick experiments or when computational resources are limited.
- `models/ginconv.py`, `gat.py`, `gat_gcn.py`, `gat_gcn_gin.py`, `gcn.py` : implementations of GINConvNet, GATNet, GAT_GCN, GAT_GCN_GIN and GCNNet.
- `cluster_eval.py` : evaluates a trained model on a cluster‑based split that prevents data leakage between structurally similar compounds (see Section 3).

### Provided models and results
Pre‑trained model weights (seed 42, single run) are available:
- `model_GINConvNet_best.model`
- `model_GATNet_best.model`
- `model_GAT_GCN_best.model`
- `model_GCNNet_best.model`
- `model_GAT_GCN_GIN_best.model`

Corresponding evaluation metrics (MSE, RMSE, MAE, Pearson, Spearman, CI, R²) on the validation set are saved in `result_<Model>_best.csv`.  
For the 10‑run statistics, see `result_<Model>_10runs.csv` and `summary_<Model>_10runs.csv`.

## Step‑by‑step running

### 0. Environment setup
Create a conda environment and install dependencies:
```bash
conda create -n geometric python=3.8
conda activate geometric
conda install -y -c conda-forge rdkit
# Install PyTorch (example for CUDA 11.7)
conda install pytorch torchvision torchaudio cudatoolkit=11.7 -c pytorch
# Install PyTorch Geometric
pip install torch-scatter torch-sparse torch-cluster torch-spline-conv -f https://data.pyg.org/whl/torch-2.0.0+cu117.html
pip install torch-geometric

```

## 1. Preprocess the data
Running
```sh
conda activate geometric
python 3Dcreate_data.py
```
This reads data/train.csv, data/val.csv and data/test.csv, generates 3D molecular graphs, and saves train.pt, val.pt and test.pt inside data/processed/.

## 2. Train a prediction model
Single‑run training (recommended for quick experiments) 
Running 

```sh
conda activate geometric
python training_single.py <model_index> [cuda_device]
```
model_index : 0 = GINConvNet, 1 = GATNet, 2 = GAT_GCN, 3 = GCNNet, 4 = GAT_GCN_GIN.
cuda_device : GPU id (default 0).
Example: python training_single.py 2 0 trains GAT_GCN on GPU 0.
The best model (lowest validation MSE) is saved as model_<Model>_best.model, and validation metrics are written to result_<Model>_best.csv.
Multi‑run training with confidence intervals
```sh
python training.py <model_index> [cuda_device] 
```
Runs the same model 10 times with different random seeds (base seed 1234 + run index).
After all runs it prints:
mean and standard deviation of all metrics.

95% confidence intervals (t‑distribution) for each metric.
Results are saved in result_<Model>_10runs.csv and summary_<Model>_10runs.csv.
You can easily change N_RUNS or BASE_SEED inside the script to adjust the number of repetitions or to test sensitivity to the random seed.

All provided pre‑trained models were obtained with training_single.py using seed 42.

## 3.Cluster‑based split evaluation
To further assess model robustness, we provide a clustering‑based evaluation that avoids training/testing on structurally similar molecules.
```sh
python cluster_eval.py --csv data/full_dataset.csv --model model_GINConvNet_best.model --model_class GINConvNet --cuda 0 
```
--csv : full dataset (must contain compound_iso_smiles and affinity columns).
--model : path to a trained .model file.
--model_class : one of GINConvNet, GATNet, GAT_GCN, GCNNet, GAT_GCN_GIN.
--cuda : GPU id (default 0).
--cutoff : Butina clustering cutoff (default 0.4). Smaller values create more clusters and a stricter split.
The script:
Performs Butina clustering on Morgan fingerprints.
Splits clusters into train / validation / test (80 / 10 / 10%).
Generates PyG .pt files for each split (cluster_train.pt, cluster_val.pt, cluster_test.pt) – these are already provided in data/processed/.
Loads the chosen model, predicts on the cluster test set, and computes metrics with 1000‑sample bootstrap confidence intervals.
Results are saved under cluster_eval_results/.
Use --skip_cluster if the .pt files already exist:
```sh
python cluster_eval.py --model model_GAT_GCN_best.model --model_class GAT_GCN --skip_cluster 
```
